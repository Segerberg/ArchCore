LIST OF STANDARDS MAPPED. Click on standard name below to go to more information about the standard.1

CDWA: The Categories for the Description of Works of Art describes the content of art databases by articulating a conceptual framework for describing and accessing information about works of art, architecture, other material culture, groups and collections of works, and related images. The CDWA includes over 500 categories and subcategories. A small subset of categories are considered core in that they represent the minimum information necessary to identify and describe a work. CDWA is the basis of CCO and the data model and cataloging rules are incorporated into CONA. Only the subset of CDWA categories that map to other standards are presented in this crosswalk.

CCO: Cataloging Cultural Objects was published by the American Library Association (ALA). CCO provides guidelines for selecting, ordering, and formatting data used to populate catalog records based on core categories in CDWA and VRA Core. CCO is a set of rules surrounding various elements from CDWA (which contains elments and rules) and VRA Core (which contains elements); it is more directly analogous to AACR and DACS.

CONA and other Getty Vocabularies: The Cultural Objects Name Authority (CONA) is a structured vocabulary containing authority records for cultural works, including architecture and movable works such as paintings, sculpture, prints, etc., whether extant, destroyed, or conceptual. See the full list of CONA fields. CONA is compliant with and an implementation of CDWA and CCO. CONA is linked to the other Getty vocabularies, ULAN, TGN, AAT, and IA for the purpose of controlling values and allowing more efficient access. For example, the creator in a CONA record is linked to the ULAN record for that artist. The Authorities categories of CDWA map to ULAN, TGN, AAT, and IA (Iconography Authority).

CIDOC CRM: The International Committee for Documentation is a committee of the of the International Council of Museums. The CIDOC Conceptual Reference Model(CRM) provides definitions and a formal structure for describing the implicit and explicit concepts and relationships used in cultural heritage documentation. The mapping here is a draft; changes may be made as Getty implements CRM in CONA and the other Getty Vocabularies.

Linked.Art: Linked.Art here refers to the shared model being devised by a community of art information shareholders working together; it is based on Linked Open Data to describe art. Linked.Art is under active development.

LIDO and CDWA Lite: The Lightweight Information Describing Objects (LIDO) schema is an XML harvesting schema is intended for delivering metadata, for use in a variety of online services, from an organization’s online collections database to portals of aggregated resources, as well as exposing, sharing and connecting data on the web. It is not intended to be used as a basis for a collection management system or to support loan and acquisition activities. CDWA Light was an XML schema to describe core records for works of art and material culture based on CDWA and CCO. CDWA Lite is no longer maintained. It CDWA Lite has now been incorporated to the newer standard, LIDO.

VRA Core: The Visual Resources Association Core categories 4.0 is an XML expression of the VRA Core metadata element set as well as an initial blueprint of how those elements should be structured.

MARC/AACR: MARC formats are standards for the representation and communication of bibliographic and related information in machine-readable form, produced by the Library of Congress. The Anglo-American Cataloguing Rules (AACR) was a national cataloging code first published in 1967. AACR2, the Second Edition, was published jointly by the American Library Association, the Canadian Library Association, and the Chartered Institute of Library and Information Professionals in the UK. Note that AACR is being replaced by Resource Description and Access(RDA) the new, unified cataloging standard designed for the digital world and an expanding universe of metadata users.

MODS: The Metadata Object Description Schema is a schema for a bibliographic element set that may be used to carry selected data from a subset of the MARC 21 records as well as to enable the creation of original resource description records.

Dublin Core: The Dublin Core Metadata Initiative is an open forum engaged in the development of interoperable online metadata standards that support a broad range of purposes and business models.

DACS: Describing Archives: A Content Standard is an output-neutral set of rules for describing archives, personal papers, and manuscript collections, and can be applied to all material types. It is the US implementation of international standards ISAD(G) and ISAAR(CPF)) for the description of archival materials and their creators.

EAD: The Encoded Archival Description Document Type Definition(DTD) is a standard for encoding archival finding aids using XML, maintained by the Library of Congress in partnership with the Society of American Archivists.

Object ID: Object ID is an international standard for describing cultural objects developed from a subset of the CDWA and in collaboration with the museum community, police and customs agencies, the art trade, insurance industry, and valuers of art and antiques.

CIMI: The Consortium for the Computer Interchange of Museum Information attribute set is part of a Z39.50 Profile for cultural heritage information.

FDA Guide: The Guide to the Description of Architectural Drawings is consistent with the format and rules of CDWA, but containing guidelines, conventions, and standards that focus specifically on architectural drawings and documents. It resulted from the work of the Architectural Drawings Advisory Group(ADAG), through the Foundation for Documents of Architecture(FDA).
